# Contributing

Thanks for your interest in improving **Secret Hitler — Pass & Play**! This is a
non-commercial, fan-made project licensed under **CC BY-NC-SA 4.0** — by
contributing, you agree your contributions are licensed under the same terms.

## Ground rules

- Be respectful — see [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
- This project is **non-commercial**. Don't add ads, paywalls, telemetry, or
  anything that monetizes the game (the Secret Hitler license forbids it).
- Don't add **official Secret Hitler artwork**. Keep all assets original.

## Project layout

```
assets/sh-engine.js   Pure rules engine (no DOM/network). The source of truth.
assets/sh-bot.js      Pure bot "brain" (AI decisions).
assets/sound.js       Synthesized SFX (WebAudio, zero asset files).
assets/ui.js          Pass-and-play DOM UI.
assets/styles.css     Theme.
assets/app.html       GENERATED by build.js — do not edit by hand.
build.js              Inlines css + engine + bot + sound + ui into app.html.
tools/make-icon.js    Zero-dependency PNG icon generator.
App.js                Expo shell: loads app.html into a react-native-webview.
tests/                Node tests (no deps): engine fuzz + headless UI smoke.
```

The engine, bot, sound, and UI are **separate modules** so the engine/bot can be
unit-tested and reused by a future networked version. `build.js` inlines them into
the single `assets/app.html` the WebView loads.

## Development

Use **Node 20** (see `.nvmrc`). Expo SDK 51's local tooling does not work reliably
on Node 22.

```bash
npm install
npm test            # engine tests + headless UI smoke test
npm run build:html  # regenerate assets/app.html after editing any source module
npm run icon        # regenerate the app icon
npm start           # build + expo start (Expo Go)
```

**Always run `npm run build:html` (or `npm test`) after editing `sh-engine.js`,
`sh-bot.js`, `sound.js`, `ui.js`, or `styles.css`** — `app.html` is generated.

## Tests are required

Every change must keep both suites green (`npm test`):

- `tests/engine.test.js` — config validation, targeted rules, and fuzz: thousands
  of full random / 3-faction / bot-vs-bot games driven to completion.
- `tests/ui.smoke.test.js` — drives complete games through the real UI action
  layer (incl. bots) against a stubbed DOM.

If you change rules, add or update targeted assertions. If you change the UI flow,
make sure the smoke test still drives it.

## Pull requests

1. Branch from `main`.
2. Keep changes focused; describe what and why.
3. `npm test` must pass; run `npm run build:html` so `app.html` is current.
4. Fill in the PR template.

## Reporting bugs / ideas

Open an issue using the templates. For anything that could break or confuse a
game, please include player count, configuration (bots? Communist mode?), and
steps to reproduce.
